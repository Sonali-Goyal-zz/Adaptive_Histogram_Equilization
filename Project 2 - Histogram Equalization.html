1. histogram(I, n, min, max)

The program takes in an input image, the number of bins, and a min and a max value. The range (min - max + 1) is equally divided by the number of bins and rounded of to the nearest integer using the ceil function. The pixel intensity values in the range are allocated to the respective bins and then divided by the total number of pixels in the bins, to get the relative frequency in every bin.

To execute the program, run the following,

./histogram_2 <path to image> <bins required> <min> <max>

The program outputs the relative frequency of intensity values in each bin.

 The code for the program can be found here, ./code/histogram/histogram.cxx.

2. histoeq(I, n, min, max)

As with histogram, this program too takes in the same parameters and outputs an image with histogram equalization performed on it. This function first calls the function histogram to create the pdf's/ relative frequency for every bin. The function histoeq then uses these pdf values to calculate the cdf for an intensity value and maps it to an intensity value between the input min and max.

To execute the program, run the following,

./histoeq <path to image> <bins required> <min> <max> 1

Here '1' is necessary parameter, as that ensures that the overloaded form of the program which performs blending operate in the same space. Check below too see how blending works.

The program outputs an image image_out.fits  with histogram equalization performed on it.

The code for the program can be found here, ./code/histoeq/histoeq.cxx.

6. Discussion

a. Crowd: The image is enhanced pretty well as people (dominant intensity and low) occupy most of the real estate on the image. Equalization, by nature, acts well on such images and in redistributing intensities.

b. University: Similar to the last image, this image is also enhanced pretty well, as most of the scene is in dark and equalization re-distributes intensities.

c. People: Enhanced pretty well. Simialr reason as with the last two images.

d. Portal: Not enhanced pretty well, though some features become evident. Intensites in the original image are distributed through out and simple equalization doesn't helps much.

e. Phobos: Not enhanced pretty well. Light features get enhanced, but still a majority of them remain in dark. The histogram is shifted to the right, but the ratio between lower and high intensity values is still high. 

f. Aerial: Enhanced pretty well. Most of the intensity values are high and the image has a washed out apperance. Equalization does a good job of redistributing intensities and bringing out the contrast.

7. Alpha blending

This program blends the original intensity value with the new intensity value (after histogram equalization) using an alpha value and outputs a blended image.To execute the program, run the following,

/histoeq <path to image> <bins requiured> <min> <max> <alpha>

where <alpha> should be between 0 and 1.  An alpha value of 0 outputs the original image and an alpha value of 1 outputs an image with simple histogram equalization.

The program outputs an image image_out.fits.

For all these images, blending helps keep equalization evident and at the same time also removes some of the noise (in the form of grain) in the image.

However for the image People, blending doesn't helps much as it still has lot of grain. In particular, lower blending values generate more soothing images (applicable to the images above).
